tl;dr: SQLite is awesome.

SQLite is awesome.

I generally choose the simplest solution to a problem until it I run into its limits. At that point I can evaluate more advanced options from a better vantage.

I've been using Firebase a lot in the past few years. I have a handful of live projects using it that were simple to setup, require almost no maintenance, and costs next to nothing. For authentication, Firebase is still my absolute go-to.

However, Firebase's storage options are limited and I knew I wanted something more powerful when I built VenueTube which would be more data intensive.

In the past I'd used MySQL and MongoDB professionally but following the principle of doing the simplest thing possible, I elected to use SQLite. There are a few things I haven't ironed out yet but overall I'd call it a smashing success.

WebAssembly is awesome, too.

One technology that I'm super excited about right now is WebAssembly. Years ago I decided to trade the power and control of C for the ubiquity of JavaScript. For the most part it has felt like the right decision and my background in C served as a solid foundation for work on the web.

That said, amidst a background of bloated web applications with huge dependency graphs, the arrival of Wasm has been a breath of fresh air. I'm sure I've encountered many web apps using Wasm under the hood but the first project to really grab my attention was SWC a Rust-based web transpiler that compiles to Wasm.

Working on developer tools at Bloomberg we were super excited about SWC. I put it to use to build the browser-only Typescript REPL in my notry-ts post
